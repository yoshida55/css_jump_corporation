# コンテンツ幅と全幅の違い

## 質問
▢　widthは全体的に文字がはいればいいのか？つまりカンプ通りでなくていい？




## 実装ルール
🎯 結論: セクションごとにしっかり作る方が圧倒的に良い
ステップ1: 共通設定だけ先に書く
ステップ2: ヘッダーを100%完成させる
ステップ3: 次のセクション(News)へ
繰り返し

## 基本ルール

| 種類 | CSS | 動作 |
|------|-----|------|
| **全幅** | `width: 100%`<br>(max-widthなし) | 画面サイズに応じてどこまでも広がる |
| **コンテンツ幅** | `max-width: 1000px`<br>`margin: 0 auto` | 1000pxで止まる。それ以上広がらない |

## Figmaの1400pxとは?

- **デザイナーの作業用キャンバスサイズ**
- 実装サイズとは関係ない
- 実際のコードでは1000pxで制限する

---

## 実装コード

### コンテンツ幅(1000px)
```css
.news_area,
.business_area,
#company_area {
  max-width: 1000px;
  margin: 0 auto;
  padding: 0 20px;
}
```
★この20pxは、通常コンテンツ幅にするのはいいが、
時によっては画面にくっついてみずらいので基本２０px とするのが多い
「仮の値」で、本来はFigma確認が必要

★margin: 0 auto;はきまり。

★全幅はかかなくてもデフォルトなので基本かかない

### 全幅
```css
/* 何も書かなくてOK(デフォルトで100%) */
.header,
.main_visual_container,
.about_area,
.footer {
  width: 100%;
}
```
